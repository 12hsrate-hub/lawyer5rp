# Редизайн страницы проверки тестов (`/exam-import-test`)

## Контекст
Текущий экран полезен как операционный/технический, но перегружен деталями для обычного пользователя. Основные улучшения: вынести итог наверх, упростить язык, скрыть тех.прозрачность на второй уровень и заменить «таблицу с деталями» на карточный сценарий проверки.

## Предлагаемая целевая структура
1. **Верхняя сводка результата**: общий статус, сколько засчитано/с замечаниями/не проверено, кнопки действия.
2. **Панель быстрых фильтров**: Все / К проверке / С замечаниями / Без замечаний.
3. **Список работ**: компактные карточки по строкам с ключевыми полями и быстрым действием.
4. **Карточка разбора работы** (drawer/modal):
   - что засчитано;
   - что не засчитано;
   - что исправить;
   - подробности по вопросу (collapsible).
5. **Технические поля** — только в блоке «Для администратора/разработчика» (collapsed по умолчанию).

## Ключевые UX-решения
- Термин «Импорт» увести на второй план; на первом — «Проверка тестов».
- Числовой итог + статус всегда в первом экране.
- Баллы показывать не только числом, но и через понятные статусы.
- Сравнение ответа: короткая причина + список различий, а не длинный текст «Пояснение» без структуры.
- «Прозрачность проверки» скрыть в accordion «Технические детали».

## Backend view-model (без rewrite)
Добавить `ui_summary` и `ui_questions` в detail payload:
- `ui_summary`: `{status_label, score_percent, correct_count, partial_count, wrong_count, needs_attention_count}`;
- `ui_questions[]`: `{question_no, status, short_reason, matched_points[], missing_points[], wrong_points[], expected_short, rationale_full}`.

Сырые `payload` и внутренние поля оставить для техрежима (`tech_payload`).

## Шаблон и фронтенд
- В `exam_import.html` добавить отдельную секцию «Итоги проверки» над таблицей.
- В detail modal заменить текущую таблицу вопросов на карточки вопросов с раскрытием.
- В JS вынести маппинг статусов и человеко-понятных текстов в единый слой форматирования.
- В CSS добавить слой визуальной иерархии: крупные KPI-карточки, спокойные нейтральные поверхности, мягкие badge-статусы.

4. Перенести технические детали в collapsed блок.
